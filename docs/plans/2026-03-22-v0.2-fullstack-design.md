# DogNeo v0.2 — Full-Stack Upgrade Design

**Date:** 2026-03-22
**Goal:** Transform DogNeo from "demo that runs" to "out-of-box tool researchers and dog owners can trust"

## Design Principles

1. **Out-of-box**: `pip install` → `dogneo demo` produces scored results with zero config
2. **Dual audience**: bioinformatics researchers (bring your VCF) + dog owners/vets (guided mode)
3. **LLM as tutor**: every pipeline step gets a plain-language explanation (like Paul Conyngham used ChatGPT)
4. **TDD**: red-green-refactor for every phase

## Phase 2 — IEDB Binding Integration

**The key to "out-of-box scored results"**

### 2.1 IEDB API Client

```python
# dogneo/core/iedb_client.py
class IEDBClient:
    MHCI_URL = "http://tools-cluster-interface.iedb.org/tools_api/mhci/"
    CACHE_DIR = Path.home() / ".dogneo" / "cache" / "iedb"

    def predict_batch(
        self,
        peptides: list[str],
        alleles: list[str],
        method: str = "recommended",
    ) -> list[BindingPrediction]:
        """Query IEDB with rate limiting and caching."""
        ...

    def _check_cache(self, peptide: str, allele: str) -> BindingPrediction | None: ...
    def _save_cache(self, prediction: BindingPrediction) -> None: ...
```

- Rate limit: 1 second between batch requests
- Cache: `~/.dogneo/cache/iedb/{hash}.json` keyed by peptide+allele
- Timeout: 120 seconds per request
- Offline detection: catch `ConnectionError` → fallback to unscored + clear warning

### 2.2 Binding Predictor Fallback Chain

```python
# In app/rank_pipeline.py
def _get_binding_predictor(tool: str) -> str:
    """Resolve binding tool with auto-fallback."""
    if tool != "auto":
        return tool
    # Priority: netmhcpan (local) → iedb (remote) → none
    if shutil.which("netMHCpan"):
        return "netmhcpan"
    if _internet_available():
        return "iedb"
    return "none"
```

## Phase 4 — LLM Step-by-Step Explanations

### 4.1 Explainer Module

```python
# dogneo/llm/explainer.py
class PipelineExplainer:
    """Generate plain-language explanations at each pipeline step."""

    def __init__(self, router: LLMRouter): ...

    def explain_variants(self, variants: list[SomaticVariant], coding: list[SomaticVariant]) -> str:
        """'Found 8 mutations, 6 affect proteins. Key genes: TP53 (tumor suppressor), BRAF (growth signal)...'"""

    def explain_peptides(self, peptides_by_variant: dict, total: int) -> str:
        """'Generated 228 candidate peptides from 6 mutations. Each peptide is 8-11 amino acids...'"""

    def explain_binding(self, candidates: list[NeoantigenCandidate]) -> str:
        """'23 peptides bind strongly (<50nM) to your dog's DLA alleles. This means...'"""

    def explain_ranking(self, top_candidates: list[NeoantigenCandidate]) -> str:
        """'#1 candidate: TP53 V245I scores highest because... This gene is known in canine cancer for...'"""

    def explain_for_owner(self, result: RankResult) -> str:
        """Full narrative for dog owners: what was found, what it means, next steps."""

    def literature_check(self, genes: list[str]) -> str:
        """Cross-reference genes against known canine cancer literature."""
```

### 4.2 Integration Points

The `run_rank_pipeline()` calls explainer at each step, storing results in `RankResult.explanations`:
```python
result.explanations["variants"] = explainer.explain_variants(variants, coding)
result.explanations["peptides"] = explainer.explain_peptides(peptides_by_variant, total)
result.explanations["binding"] = explainer.explain_binding(candidates)
result.explanations["ranking"] = explainer.explain_ranking(ranked[:10])
result.explanations["summary"] = explainer.explain_for_owner(result)
```

CLI prints these if `--llm-tier` is not `none`. UI displays them in sidebars/expandable sections.

## Phase 5 — mRNA Designer

**Replicates what Grok did for Paul Conyngham**

### 5.1 Core Module

```python
# dogneo/core/mrna_designer.py
CANINE_CODON_TABLE: dict[str, str]  # Optimal codons for Canis lupus familiaris

@dataclass
class MRNAConstruct:
    """Complete mRNA construct ready for synthesis."""
    five_prime_utr: str        # 5' UTR sequence
    signal_peptide: str        # Optional signal peptide
    antigen_cassette: str      # Codon-optimized multi-epitope
    linker_sequences: list[str]  # Between epitopes (e.g., GPGPG, AAY)
    three_prime_utr: str       # 3' UTR sequence
    poly_a_length: int = 120   # Poly-A tail length

    @property
    def full_sequence(self) -> str: ...
    def to_fasta(self) -> str: ...
    def to_genbank(self) -> str: ...

def codon_optimize(protein_seq: str, species: str = "canine") -> str:
    """Reverse-translate protein to codon-optimized mRNA."""

def design_construct(
    candidates: list[NeoantigenCandidate],
    top_n: int = 10,
    llm_router: LLMRouter | None = None,
) -> MRNAConstruct:
    """Design multi-epitope mRNA construct from top candidates."""
```

### 5.2 LLM Role

LLM assists with:
- Reviewing the construct design for potential issues
- Suggesting optimal linker sequences between epitopes
- Explaining the design rationale in plain language
- Flagging known stability concerns

The core codon optimization is algorithmic (lookup table), NOT LLM-generated — determinism matters for reproducibility. LLM adds commentary and review.

### 5.3 CLI

```bash
dogneo design-mrna \
  --candidates results/candidates.json \
  --top-n 10 \
  --output-dir mrna_design/ \
  --llm-tier cli
```

Output:
```
mrna_design/
├── construct.fasta      # Full mRNA sequence
├── construct.gb         # GenBank format with annotations
├── design_report.html   # LLM-assisted design rationale
└── epitopes.tsv         # Selected epitopes with scores
```

- Print / export button

**Page 5 — mRNA Designer**

1. `pip install "dogneo[all]"` → `dogneo demo` → produces scored candidates with real binding affinities
2. `dogneo ui --demo` → opens browser with interactive dashboard showing ranked candidates
3. `dogneo design-mrna --candidates results/candidates.json` → outputs mRNA FASTA + GenBank
